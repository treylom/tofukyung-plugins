# Lesson Engine — 인터랙티브 진행 가이드

강의 3-Phase 흐름의 상세 지침이다.
SKILL.md의 Step 3에서 이 파일을 참조한다.

---

## Phase 1: Opening (시작)

### 1.1 인사 멘트

수강생 이름을 사용한 개인화 인사로 시작한다.

```
안녕하세요, {이름}님! 오늘 {과정명} {주차}주차 수업을 시작하겠습니다.
```

수강생 프로필에 회사/부서 정보가 있으면:
```
{회사} {부서}에서 오셨군요. 오늘도 잘 부탁드립니다!
```

### 1.2 오늘 학습 목표 안내

커리큘럼 파일에서 학습 목표를 추출하여 안내한다.

```
오늘은 이런 내용을 배웁니다:
• [목표 1]
• [목표 2]
• [목표 3]

총 [N]교시로 진행됩니다.
```

### 1.3 준비 확인

AskUserQuestion으로 준비 상태를 확인한다.

```
시작 전 확인 사항:
□ Claude Code가 열려 있나요?
□ [과정별 필요 도구]가 준비됐나요?

준비되면 "준비됐어요" 또는 "시작"이라고 말씀해 주세요.
```

수강생이 준비 안 됐다고 하면:
- 설치/실행 방법 안내
- 준비 완료 확인 후 Phase 2로 이동

## Phase 2: Lesson Progress (진행)

### 2.1 교시 시작 안내

각 교시 시작 시:
```
━━━━━━━━━━━━━━━━━━━━━━━
📚 {N}교시: {교시 제목}
예상 소요: {소요 시간}
━━━━━━━━━━━━━━━━━━━━━━━

오늘 이 교시에서는 {교시 목표}를 익힙니다.
```

### 2.2 실습 안내 패턴

실습마다 아래 패턴을 반복한다.

**Step A: 맥락 설명 (30초 이내)**
```
지금부터 {실습 목표}를 해봅니다.
{1줄 개념 설명}
```

**Step B: 명령어/작업 제시**
```
아래 명령어를 {도구}에 입력해 보세요:

{명령어 또는 작업 지시}
```

- 코드 블록으로 감싸서 명확하게 제시
- 한 번에 하나씩만 제시 (두 개 이상이면 순서 명시)
- `Copy` 버튼이 보이도록 마크다운 코드 펜스 사용

**Step C: 진행 확인**
AskUserQuestion 사용:
```
입력해 보셨나요? 어떻게 됐는지 알려주세요!
(잘 됐으면 "됐어요" / 에러 나면 에러 메시지 복붙)
```

**Step D: 결과 확인 분기**

| 응답 | 다음 행동 |
|------|----------|
| "됐어요" / "성공" | 칭찬 + 결과 의미 설명 + Step A로 |
| 에러 메시지 복붙 | 트러블슈팅 (2.3) |
| "잘 모르겠어요" | 한 단계 더 상세히 안내 |
| "건너뛰어도 돼요?" | 이유 확인 후 판단 |

### 2.3 트러블슈팅

에러 발생 시 단계별로 접근한다.

**Level 1: 흔한 에러 체크리스트**
```
아, 이 에러는 자주 나와요. 아래를 확인해 볼게요:
□ [가장 흔한 원인 1]
□ [가장 흔한 원인 2]
□ [가장 흔한 원인 3]

어떤 걸 확인해 보셨나요?
```

**Level 2: 에러 메시지 해석**
```
에러 메시지를 같이 읽어볼게요.
"{에러 핵심 부분}" — 이건 {설명}을 의미해요.
{해결 방법}을 시도해 보세요.
```

**Level 3: 직접 확인 (Bash 사용)**
```bash
# 환경 상태 확인
{관련 명령어}
```
확인 결과를 수강생에게 설명하고 다음 단계 안내.

**Level 4: 에스컬레이션**
3번 시도 후에도 해결 안 되면:
```
이건 환경 설정 문제일 수 있어요.
잠깐 강사님께 확인 요청을 드릴게요.
[혹은 대안 경로 제시]
```

### 2.4 숙련도별 안내 조정

수강생 프로필의 `ai_level`에 따라 안내 깊이를 조정한다.

| 레벨 | 안내 방식 |
|------|----------|
| `beginner` | 명령어 한 줄씩 + 각 단어/옵션 의미 설명 + 비유 활용 |
| `intermediate` | 명령어 + 핵심 포인트만 설명 + "왜 이렇게 하는지" 포함 |
| `advanced` | 목표만 제시 → 수강생이 방법 선택 → 결과 확인 |

### 2.5 교시 전환 멘트

교시 완료 시:
```
잘 하셨어요! {N}교시 완료입니다. 🎉

방금 {핵심 성취 요약}을 해봤어요.

잠깐 쉬고 싶으면 쉬세요. 준비되면 "다음 교시 시작"이라고 말씀해 주세요.
```

수강생이 "다음" / "계속" / "응" 등 응답하면 다음 교시 Step A로.

## Phase 3: Closing (마무리)

### 3.1 학습 요약

```
오늘 수업 마무리할게요! 잠깐 돌아볼게요.

오늘 배운 것들:
✅ {교시 1에서 배운 핵심}
✅ {교시 2에서 배운 핵심}
✅ {교시 3에서 배운 핵심}
```

### 3.2 GitHub 저장/푸시 (해당하는 경우)

커리큘럼 파일에 GitHub 작업이 포함된 경우:

```
오늘 작업한 내용을 저장해둘게요.

1단계 — 변경사항 저장:
git add .
git commit -m "feat: {오늘 날짜} {주차} 실습 완료"

입력해 보셨나요?
```

완료 확인 후:
```
2단계 — GitHub에 올리기:
git push

올라가면 GitHub에서 확인해보세요!
```

### 3.3 다음 주차 예고

```
다음 주차에는 {다음 주차 주제}를 배울 거예요.
오늘 배운 {현재 주차 핵심 개념}을 기반으로 더 발전시켜나갑니다.

궁금한 점 있으면 언제든지 물어보세요!
수고하셨습니다, {이름}님 👏
```

### 3.4 과제 안내 (커리큘럼 파일에 과제가 있는 경우)

```
📌 오늘의 과제:
{과제 내용}

마감: {마감일 또는 "다음 수업 전"}
```

### 3.5 프로필 업데이트

Closing 완료 후 수강생 프로필을 업데이트한다.

```json
{
  "completed_lessons": ["기존 목록" + "{과정ID}-{주차}주차"],
  "struggles": ["기존 목록" + "이번 세션에서 어려워했던 것"],
  "strengths": ["기존 목록" + "이번 세션에서 잘 한 것"]
}
```

## 수강생 응답 패턴 핸들링

| 수강생 말 | 해석 | 대응 |
|----------|------|------|
| "됐어요" / "됐다" / "성공" | 실습 성공 | 칭찬 + 다음 단계 |
| "안됐어요" / "에러" / 에러 메시지 | 실습 실패 | 트러블슈팅 |
| "잘 모르겠어요" / "?" | 이해 불가 | 다른 방식으로 재설명 |
| "건너뛰어도 돼요?" | 선택적 스킵 요청 | 이유 묻고 판단 |
| "잠깐만요" / "잠시만" | 처리 중 | 기다리기 |
| "다 됐어요" / "끝났어요" | 전체 완료 | Closing으로 이동 |
| "이게 뭐예요?" | 개념 질문 | 간단히 설명 후 계속 |
| "왜요?" / "왜 이렇게 해요?" | 이유 질문 | 배경 설명 후 계속 |
| "어려워요" | 어려움 호소 | 더 세분화하여 안내 |
